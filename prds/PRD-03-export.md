# PRD-03 · Export System

**Feature:** JSON download and Markdown clipboard copy of the full intel package
**Status:** Shipped v1
**Owner:** Foxworks Studios

---

## Problem

The intel package generated by ProspectFold is valuable beyond the app itself. A founder needs to:
- Share the ICP with a copywriter writing cold email sequences
- Paste the angles into a pitch deck or Notion doc
- Archive the research for a vertical so it can be referenced months later
- Feed structured data into another tool (CRM importer, Zapier, a spreadsheet)

Without export, the intel is trapped in the app and the localStorage history log.

---

## Export Formats

### Format 1: JSON Download (`↓ JSON`)

Downloads a `.json` file with a metadata wrapper around the raw intel package.

**Filename:** `prospect-{naicsCode}-{YYYY-MM-DD}.json`
Example: `prospect-541511-2026-03-05.json`

**File structure:**
```json
{
  "meta": {
    "version": 1,
    "naicsCode": "541511",
    "naicsLabel": "Custom Software Development",
    "generated": "2026-03-05T20:00:00.000Z"
  },
  "summary": "...",
  "icp": {
    "company_types": [...],
    "company_sizes": [...],
    "qualifying_criteria": [...],
    "signals": [...]
  },
  "angles": [...],
  "searches": {
    "apollo": [...],
    "google": [...],
    "linkedin": [...]
  },
  "qualification_checklist": [...],
  "red_flags": [...],
  "enrichment_urls": [...]
}
```

**Implementation:** `URL.createObjectURL(Blob)` → programmatic `<a>` click → `URL.revokeObjectURL()`
No Electron file dialog needed — browser download API works within Electron renderer.

**Use cases:**
- Long-term archival of research by vertical
- Feeding into downstream tools (Zapier, Make, n8n webhook)
- Version-controlled intel library in a git repo
- Importing into a future ProspectFold saved packages feature

---

### Format 2: Markdown Copy (`⊞ Copy Markdown`)

Writes a fully formatted markdown document to the clipboard. No file is created — intended for pasting into Notion, Linear, Slack, a pitch deck speaker notes, or a cold email brief.

**Markdown structure:**
```markdown
# Prospect Intel: {naicsLabel} ({naicsCode})
_{Generated date}_

## Summary
{summary}

## Company Types
- {type}
- ...

## Qualifying Criteria
- [ ] {criterion}
- ...

## Buying Signals
- {signal}
- ...

## Angles
### 1. {name}
{hypothesis}
> "{hook}"

### 2. ...

## Apollo Searches
### {label}
`{query}`
_{industry: ... · employee_count: ... · keywords: ...}_

## Qualification Checklist
- [ ] **{criterion}** — {how_to_verify}

## Red Flags
- ❌ {flag}
```

**Implementation:** `navigator.clipboard.writeText(markdown)` — works in both browser and Electron renderer.

## UI

Both export controls appear as a row of buttons **above the ICP Summary Banner**, right-aligned:

```
[↓ JSON]  [⊞ Copy Markdown]
─────────────────────────────────────────────
ICP SUMMARY
...
```

**`↓ JSON` button:**
- Default: `border: T.border`, `color: T.textSub`
- Hover: `border: T.accent`, `color: T.accent`
- No loading state — download is synchronous

**`⊞ Copy Markdown` button:**
- Uses `CopyButton` component
- Default: `COPY` (for inline use) / `⊞ Copy Markdown` (in export row)
- Success state: `✓ Copied!` for 1.8 seconds, green styling
- `CopyButton` accepts `label` and `successLabel` props

## What's Included in Each Export

| Section | JSON | Markdown |
|---------|------|----------|
| Meta (version, naicsCode, generated) | ✓ | ✗ (just date in heading) |
| Summary | ✓ | ✓ |
| Company types | ✓ | ✓ |
| Company sizes | ✓ | ✗ (rarely useful out of context) |
| Qualifying criteria | ✓ | ✓ (as checkbox list) |
| Buying signals | ✓ | ✓ |
| Angles (name + hypothesis + hook) | ✓ | ✓ |
| Apollo searches + filters | ✓ | ✓ |
| Google searches | ✓ | ✗ (clutters markdown) |
| LinkedIn searches | ✓ | ✗ (clutters markdown) |
| Qualification checklist | ✓ | ✓ (as checkbox list) |
| Red flags | ✓ | ✓ |
| Enrichment URLs | ✓ | ✗ (links don't render in all paste targets) |

## Future Improvements

- **PDF export** — formatted PDF with Foxworks branding; suitable for client-facing or partner sharing
- **Selective export** — checkboxes to include/exclude specific sections before exporting
- **Export to Notion** — direct Notion API integration (requires OAuth; post-MVP)
- **Webhook output** — POST the JSON payload to a configurable URL on generation (Zapier, Make, n8n)
- **Batch export** — export multiple history entries as a single JSON array or zip of individual files
- **Named exports** — allow the user to name a run before exporting ("Q1 Fintech Campaign") so filenames are meaningful
